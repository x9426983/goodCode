链路全景

① 请求入口
RiverServiceImpl.recall(Request)
└─ 创建 ContextBase，内含新的 RiverReplayLog 实例

② 图执行阶段（DAG 算子逐层处理）
├─ Recaller 算子（30+ 个）
│   └─ ReplayLogUtil.recordRecallResult() → replayLog.originRecalls
│
├─ Logger 算子（专职记录中间数据）
│   ├─ ReplayLogByRecallList / ReplayLogByRecallSource → extendData
│   ├─ WmIntentEnvReplayLogWriter（环境感知数据）→ extendData
│   ├─ GeneralReplayLogPacker（组装结果）→ extendData
│   └─ LogStrategyResultToExtInfo / TableFieldsMapLog 等 → extendData
│
└─ 其他算子（Fetcher/Scorer 等）
└─ 按需嵌入 ReplayLogUtil 调用，记录中间结果

③ 日志落盘（异步，不阻塞主流程）
RiverServiceImpl.recordReplayLog(contextBase)
├─ 黑名单检查（replay_log_scene_blacklist 配置的 scene 跳过）
├─ 初始化基本信息（request/response/interfereLog 写入）
├─ 转换为 XMDLog 格式（convert2logString）
├─ 大小检查（>= 1MB 上报 CAT 告警）
├─ 写入主日志 → logger_wmrec_river_replaylog
└─ [可选] 重放流量 && 白名单 scene → 额外写入 message 日志

---
数据结构

RiverReplayLog {
基础字段: userId, uuid, bizTraceId, sceneId, timestamp, 地理位置 ...
originRecalls:  Map<召回源名, List<商品ID>>    ← Recaller 阶段写入
userTriggers:   Map<key, Object>              ← 用户行为触发数据
extendData:     Map<key, Object>              ← 所有中间过程数据（最主力）
message:        Map<key, Object>              ← 重放流量大数据量专用
request/response:  JSON 字符串               ← 原始请求与最终响应
}

关键控制点

┌──────────────┬──────────────────────────────────────────────────────────────┐
│    控制点    │                             机制                             │
├──────────────┼──────────────────────────────────────────────────────────────┤
│ 场景过滤     │ replay_log_scene_blacklist 配置，命中则整个请求跳过日志      │
├──────────────┼──────────────────────────────────────────────────────────────┤
│ 重放流量标记 │ 请求 extraParams 中 resetFlag=1，触发额外的 message 日志写入 │
├──────────────┼──────────────────────────────────────────────────────────────┤
│ 性能保护     │ 日志超 1MB 触发 CAT 告警；落盘异步执行不阻塞响应             │
├──────────────┼──────────────────────────────────────────────────────────────┤
│ 数据分离     │ 大数据量走 recordMap2Message()，避免主日志膨胀               │
└──────────────┴──────────────────────────────────────────────────────────────┘
