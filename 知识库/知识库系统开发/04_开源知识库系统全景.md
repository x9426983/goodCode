# 开源知识库系统全景

> 研究日期：2026-06-25

---

## 一、个人知识管理工具

### Obsidian ★★★★★（最推荐）

**定位：** 本地优先的 Markdown 知识库，双向链接构建知识图谱

```
Obsidian 核心能力
├── 存储：纯本地 Markdown 文件，数据完全自主
├── 双向链接：[[文档名]] 自动建立双向引用
├── 图谱视图：可视化展示文档之间的关系网络
├── 插件生态：1000+ 社区插件
│   ├── Dataview：用 SQL 查询笔记内容
│   ├── Templater：模板自动化
│   ├── Smart Connections：本地 AI + Embedding 检索
│   └── Git：自动同步到 GitHub
└── 付费功能：Obsidian Sync（云同步）、Publish（发布为网站）
```

**适合场景：** 个人技术笔记、读书笔记、知识图谱构建

**与 RAG 集成：**
- 本地 Ollama + Obsidian Smart Connections 插件 = 本地知识库问答
- 文件全部是 Markdown，天然适合作为 RAG 的原始文档来源

---

### Logseq

**定位：** 大纲式笔记 + 双向链接，本地优先

```
特点：
├── 块（Block）是最小单元，而非页面
├── 每个块可以独立引用和双向链接
├── 日记（Journal）驱动，适合 GTD 工作流
└── 本地存储（Markdown 或 EDN 格式）
```

**适合场景：** 研究人员、GTD 工作流、日记 + 知识管理结合

---

## 二、团队 Wiki / 文档系统

### Outline ★★★★（团队首选）

**GitHub Stars：** ~28k  
**技术栈：** React + Node.js + PostgreSQL

```
Outline 核心能力
├── 现代化 UI，接近 Notion 的编辑体验
├── 层级文档结构（Workspace → Collection → Document）
├── Markdown 原生支持 + 富文本编辑器
├── 权限管理：团队、文档级别的精细权限
├── 全文搜索（内置 Postgres 全文检索）
├── API 完善，支持第三方集成
└── 私有部署（Docker，10 分钟部署完毕）
```

**适合场景：** 中小团队技术 Wiki，Confluence 的开源替代

---

### BookStack

**GitHub Stars：** ~15k  
**技术栈：** PHP + Laravel + MySQL

```
内容层级：Shelf（书架）→ Book（书）→ Chapter（章）→ Page（页）
特点：
├── 层级最清晰，适合有大量结构化文档的团队
├── WYSIWYG 编辑器（对非技术人员友好）
├── 内置搜索、版本历史、PDF 导出
└── 轻量级，服务器要求低
```

**适合场景：** 运营文档、产品文档、需要严格层级管理的知识库

---

### Wiki.js

**GitHub Stars：** ~24k  
**技术栈：** Node.js + Vue.js + 多数据库后端

```
Wiki.js 特色
├── 支持多种存储后端：Git、数据库、云存储
├── 多种编辑器：Markdown、WYSIWYG、AsciiDoc、代码
├── 完善的权限系统（基于 RBAC）
├── 内置 Elasticsearch 集成（全文搜索）
└── 支持 LDAP / SSO 统一认证
```

**适合场景：** 需要 Git 存储文档的团队（文档版本管理强需求）

---

### Docusaurus（Meta 开源）

**GitHub Stars：** ~55k  
**技术栈：** React + MDX

```
定位：技术文档网站生成器（非 Wiki）

核心特点：
├── Markdown/MDX 驱动
├── 版本化文档（支持多版本并行维护）
├── 插件化：algolia 搜索、i18n、PWA
└── 与 GitHub Pages / Vercel 集成一键部署
```

**适合场景：** 开源项目文档、API 文档、对外的技术知识库

## 三、AI 知识库专用平台（2023-2025 新兴，★ 重点关注）

### Dify ★★★★★（国内最流行）

**GitHub Stars：** ~100k+  
**定位：** 可视化 LLM 应用开发平台，内置知识库 RAG

```
Dify 知识库能力
├── 知识库管理
│   ├── 上传文档（PDF/Word/Markdown/网页）
│   ├── 自动分块（多种策略可选）
│   ├── Embedding 模型选择（OpenAI/国内各大模型/本地）
│   └── 向量数据库：Weaviate / Qdrant / Milvus / pgvector
├── 应用编排
│   ├── 聊天机器人（问答式知识库助手）
│   ├── Agent（工具调用 + 知识库联合）
│   └── Workflow（复杂多步骤流程）
└── 部署方式
    ├── SaaS（dify.ai）
    └── 私有部署（Docker Compose，官方文档完善）
```

**适合场景：** 企业快速搭建内部知识库问答，不想从零写代码

---

### RAGFlow ★★★★（文档解析最强）

**GitHub Stars：** ~40k+  
**机构：** InfiniFlow（国内）  
**核心差异化：深度文档解析能力**

```
RAGFlow 最强能力：文档理解
├── 表格识别：保留表格结构，不丢行列关系
├── 版式分析：识别标题层级、多栏布局
├── 图片 OCR：提取图片中的文字内容
├── 数学公式：LaTeX 格式保留
└── 视觉理解：图表、流程图的语义理解

vs 普通 RAG 工具的区别：
  普通工具：PDF → 纯文本 → 表格变乱码
  RAGFlow ：PDF → 结构化内容 → 表格完整保留
```

**适合场景：** 金融报告、法律文件、技术手册等格式复杂的文档问答

---

### FastGPT

**GitHub Stars：** ~20k+  
**定位：** 知识库问答专用，上手最简单

```
特点：
├── 对话流程可视化配置（类似流程图编辑）
├── 知识库管理简单（上传即可用）
├── 支持工作流（问题分类 → 知识检索 → 回答）
└── 私有部署（Docker，15 分钟起）
```

**适合场景：** 快速验证知识库问答场景，个人/小团队

---

### AnythingLLM

**GitHub Stars：** ~35k+  
**定位：** All-in-one 私有 AI 助手

```
特点：
├── 桌面应用（Mac/Windows/Linux）+ 服务端
├── 支持所有主流 LLM（本地 Ollama / 云端 API 均可）
├── 知识库上传（文档/网页/代码）
├── Agent 功能（联网搜索、代码执行）
└── 完全离线可用（Ollama + 本地模型）
```

**适合场景：** 追求数据隐私、完全本地化部署的个人/小团队

## 四、各工具选型总结

```
选型决策矩阵

场景                          推荐工具
──────────────────────────────────────────────────────
个人技术笔记 + 知识图谱        Obsidian
团队技术 Wiki（替代 Confluence）Outline
对外技术文档网站               Docusaurus
企业内部知识库问答（快速上线）  Dify
文档格式复杂（表格/PDF）       RAGFlow
完全本地化，数据不出网          AnythingLLM + Ollama
个人快速搭知识库问答           FastGPT
```

## 五、组合方案推荐

### 方案一：个人技术知识库（零成本）
```
Obsidian（写作+组织）
    ↓ Git 同步
GitHub 私有仓库（版本管理）
    ↓ 触发
Dify 知识库（RAG 问答）
    → 本地问答：AnythingLLM + Ollama
```

### 方案二：小团队知识库（低成本）
```
Outline（团队 Wiki）
    + Dify（AI 问答层，接入 OpenAI API）
    + Meilisearch（全文搜索）
    全部 Docker 私有部署
```

### 方案三：企业级知识库（完整方案）
```
飞书/语雀（文档管理层）
    ↓ API 同步
Milvus（向量数据库）+ Elasticsearch（全文索引）
    ↓ 混合检索
RAGFlow（文档解析 + RAG 引擎）
    ↓ 
自研或 Dify 编排的 AI 助手
    + 权限系统（ACL 元数据过滤）
```
