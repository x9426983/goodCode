## 🗺 核心结论（一眼看懂）

```
知识库演进路径
第一代（2010s）      第二代（2018-2022）    第三代（2023至今）

Wiki / Confluence  →  Notion-style      →  AI-Native KB
  文档存储               块级结构              RAG 问答
  全文检索               双向链接              知识溯源
  手动组织               知识图谱              自动摘要
```

**最值得关注的 3 个趋势：**
1. **GraphRAG**：知识图谱 + 向量混合检索，微软开源，国内大厂跟进
2. **权限感知 RAG**：企业落地最难点，ACL 过滤是主流解法
3. **Agent 驱动知识管理**：AI 不只是"被问"，而是主动维护知识库

**最务实的个人知识库组合：**
```
Obsidian（本地知识图谱）
    + Dify / RAGFlow（私有 RAG 问答层）
    + Git 仓库（版本管理 + 代码文档同位）
```
