# 队列Queue
入队（Enqueue）：将元素添加到队列的末尾。
出队（Dequeue）：将元素从队列的开头移除。

## Queue接口
在 Java 中，Queue 是一个接口，它遵循 FIFO（先进先出）原则
`Queue` 是 `java.util` 包中的接口，继承自 `Collection` 接口，主要提供了队列的基本操作。

```java
public interface Queue<E> extends Collection<E> {
    boolean add(E e);// 添加元素，失败时抛出异常
    boolean offer(E e);// 添加元素，失败时返回 false
    E remove();// 移除并返回队头，空队列时抛出异常
    E poll();// 移除并返回队头，空队列时返回 null
    E element();// 查看队头，空队列时抛出异常
    E peek();// 查看队头，空队列时返回 null
}
```
### 主要实现类

| **实现类**                    | **特点**                        |
|----------------------------|-------------------------------|
| **LinkedList**             | 双向链表实现，可用作队列（也实现了 Deque 接口）   |
| **ArrayDeque**             | 基于循环数组的双端队列，性能通常优于 LinkedList |
| **PriorityQueue**          | 优先级队列（非 FIFO，按优先级排序）          |
| **ConcurrentLinkedQueue**  | 线程安全的无界非阻塞队列                  |
| **ArrayBlockingQueue**     | 线程安全的有界阻塞队列（基于数组）             |

#### ArrayDeque
数据结构是循环数组
```java
// 初始化（默认容量16，必须是2的幂）
public ArrayDeque() {
    elements = new Object[16];
}
// 入队（队尾添加）
public boolean offer(E e) {
    addLast(e);  // 调用 addLast
}
public void addLast(E e) {
    if (e == null) throw new NullPointerException();
    elements[tail] = e;
    if ((tail = (tail + 1) & (elements.length - 1)) == head) // 循环索引
        doubleCapacity();  // 扩容
}
// 循环索引的关键：当容量是2的幂时
// (tail + 1) & (elements.length - 1) 等价于 (tail + 1) % elements.length
// 但位运算更快
// 扩容机制
private void doubleCapacity() {
    int p = head;
    int n = elements.length;
    int r = n - p;
    int newCapacity = n << 1;  // 容量翻倍
    Object[] a = new Object[newCapacity];
    System.arraycopy(elements, p, a, 0, r);  // 复制从head到数组末尾的部分
    System.arraycopy(elements, 0, a, r, p);  // 复制从数组开始到head的部分
    elements = a;
    head = 0;
    tail = n;
}
```
#### PriorityQueue
数据结构是二叉堆
```java
// 添加元素
public boolean offer(E e) {
    if (e == null) throw new NullPointerException();
    modCount++;
    int i = size;
    if (i >= queue.length)
        grow(i + 1);  // 扩容
    size = i + 1;
    if (i == 0)  // 第一个元素
        queue[0] = e;
    else
        siftUp(i, e);  // 上浮调整
    return true;
}

// 上浮调整（最小堆）
private void siftUp(int k, E x) {
    if (comparator != null)
        siftUpUsingComparator(k, x);
    else
        siftUpComparable(k, x);
}

// 使用 Comparable 的上浮
private void siftUpComparable(int k, E x) {
    Comparable<? super E> key = (Comparable<? super E>) x;
    while (k > 0) {
        int parent = (k - 1) >>> 1;  // 父节点索引
        Object e = queue[parent];
        if (key.compareTo((E) e) >= 0)  // 如果比父节点大，停止上浮
            break;
        queue[k] = e;  // 父节点下沉
        k = parent;    // 继续向上比较
    }
    queue[k] = key;    // 找到合适位置
}

// 移除队头
public E poll() {
    if (size == 0)
        return null;
    int s = --size;
    modCount++;
    E result = (E) queue[0];  // 堆顶元素
    E x = (E) queue[s];       // 最后一个元素
    queue[s] = null;
    if (s != 0)
        siftDown(0, x);       // 下沉调整
    return result;
}

// 下沉调整
private void siftDown(int k, E x) {
    if (comparator != null)
        siftDownUsingComparator(k, x);
    else
        siftDownComparable(k, x);
}

private void siftDownComparable(int k, E x) {
    Comparable<? super E> key = (Comparable<? super E>) x;
    int half = size >>> 1;  // 非叶子节点数量
    while (k < half) {
        int child = (k << 1) + 1;  // 左孩子
        Object c = queue[child];
        int right = child + 1;     // 右孩子
        if (right < size && 
            ((Comparable<? super E>) c).compareTo((E) queue[right]) > 0)
            c = queue[child = right];  // 取较小的孩子
        if (key.compareTo((E) c) <= 0)  // 如果比孩子小，停止下沉
            break;
        queue[k] = c;  // 孩子上浮
        k = child;     // 继续向下比较
    }
    queue[k] = key;    // 找到合适位置
}
```

#### QueueTraversalUtils
QueueTraversalUtils 类提供了一些静态方法用于遍历队列
```java
/**
 * 队列遍历工具类
 */
public final class QueueTraversalUtils {
    
    private QueueTraversalUtils() {
        // 工具类，防止实例化
    }
    /**
     * 安全的FIFO遍历（不修改原队列）
     */
    public static <T> void traverseFifo(Queue<T> queue, Consumer<T> processor) {
        Objects.requireNonNull(queue, "队列不能为空");
        Objects.requireNonNull(processor, "处理器不能为空");
        
        Queue<T> copy = createCopy(queue);
        while (!copy.isEmpty()) {
            processor.accept(copy.poll());
        }
    }
    /**
     * 获取FIFO顺序的列表
     */
    public static <T> List<T> toFifoList(Queue<T> queue) {
        List<T> result = new ArrayList<>();
        traverseFifo(queue, result::add);
        return result;
    }
    /**
     * 分批处理
     */
    public static <T> void batchProcess(Queue<T> queue, int batchSize, Consumer<List<T>> batchProcessor) {
        List<T> batch = new ArrayList<>(batchSize);
        Queue<T> copy = createCopy(queue);
        
        while (!copy.isEmpty()) {
            batch.add(copy.poll());
            
            if (batch.size() == batchSize) {
                batchProcessor.accept(new ArrayList<>(batch));
                batch.clear();
            }
        }
        
        if (!batch.isEmpty()) {
            batchProcessor.accept(new ArrayList<>(batch));
        }
    }
    /**
     * 查找并处理第一个匹配的元素
     */
    public static <T> Optional<T> findAndProcess(Queue<T> queue, Predicate<T> predicate, Consumer<T> processor) {
        Queue<T> copy = createCopy(queue);
        
        while (!copy.isEmpty()) {
            T element = copy.poll();
            if (predicate.test(element)) {
                processor.accept(element);
                return Optional.of(element);
            }
        }
        
        return Optional.empty();
    }
    /**
     * 并行处理队列元素
     */
    public static <T> void parallelProcess(Queue<T> queue, Consumer<T> processor, int parallelism) {
        List<T> snapshot = new ArrayList<>(queue);
        
        ForkJoinPool pool = new ForkJoinPool(parallelism);
        try {
            pool.submit(() -> 
                snapshot.parallelStream().forEach(processor)
            ).get();
        } catch (InterruptedException | ExecutionException e) {
            Thread.currentThread().interrupt();
            throw new RuntimeException("并行处理失败", e);
        } finally {
            pool.shutdown();
        }
    }
    /**
     * 创建队列副本
     */
    @SuppressWarnings("unchecked")
    private static <T> Queue<T> createCopy(Queue<T> queue) {
        if (queue instanceof LinkedList) {
            return new LinkedList<>(queue);
        } else if (queue instanceof ArrayDeque) {
            return new ArrayDeque<>(queue);
        } else if (queue instanceof PriorityQueue) {
            return new PriorityQueue<>(queue);
        } else if (queue instanceof ConcurrentLinkedQueue) {
            return new ConcurrentLinkedQueue<>(queue);
        } else {
            // 通过反射尝试调用复制构造函数
            try {
                return (Queue<T>) queue.getClass()
                    .getConstructor(Collection.class)
                    .newInstance(queue);
            } catch (Exception e) {
                // 回退到通用方法
                Queue<T> copy = new LinkedList<>();
                copy.addAll(queue);
                return copy;
            }
        }
    }
    /**
     * 统计信息
     */
    public static class Stats {
        public final long totalElements;
        public final long processed;
        public final long failed;
        public final long skipped;
        
        public Stats(long totalElements, long processed, long failed, long skipped) {
            this.totalElements = totalElements;
            this.processed = processed;
            this.failed = failed;
            this.skipped = skipped;
        }
    }
    /**
     * 带统计的遍历
     */
    public static <T> Stats traverseWithStats(Queue<T> queue, Predicate<T> filter, Consumer<T> processor) {
        long total = queue.size();
        long processed = 0;
        long failed = 0;
        long skipped = 0;
        
        Queue<T> copy = createCopy(queue);
        
        while (!copy.isEmpty()) {
            T element = copy.poll();
            
            if (filter.test(element)) {
                try {
                    processor.accept(element);
                    processed++;
                } catch (Exception e) {
                    failed++;
                    System.err.println("处理失败: " + e.getMessage());
                }
            } else {
                skipped++;
            }
        }
        return new Stats(total, processed, failed, skipped);
    }
}
```

### **实际应用场景**
1. **任务调度**：线程池中的任务队列
2. **消息传递**：消息中间件中的消息队列
3. **广度优先搜索（BFS）**：算法中常用队列
4. **缓冲区**：生产者和消费者之间的缓冲
5. **打印队列**：管理打印任务

### **考虑与建议**
性能与其他角度
- **LinkedList**：插入/删除 O(1)，但内存开销较大（节点对象）
- **ArrayDeque**：插入/删除 O(1)，内存连续，通常性能更好
- **PriorityQueue**：插入 O(log n)，删除 O(log n)
- **阻塞队列**：适用于多线程场景，提供线程安全操作

- 需要 **FIFO 队列**：优先考虑 **ArrayDeque**（单线程）或 **ConcurrentLinkedQueue**（多线程）
- 需要 **优先级排序**：使用 **PriorityQueue**
- 需要 **线程安全阻塞操作**：使用 **ArrayBlockingQueue** 或 **LinkedBlockingQueue**
- 需要 **双端操作**：使用 **ArrayDeque**（实现了 Deque 接口）
