# 什么是装饰器模式
装饰器模式的核心：用同一个接口包裹另一个实现，在不改变原对象的前提下，透明地增强其功能，提供比继承更灵活的扩展方式。

         │ execute()│

# 示例
```java
// 饮料接口
public interface Beverage {
    String getDescription();
    double cost();
}

// 具体饮料：浓缩咖啡
public class Espresso implements Beverage {
    @Override
    public String getDescription() {
        return "浓缩咖啡";
    }
    
    @Override
    public double cost() {
        return 25.0;
    }
}

// 装饰器：调料
public abstract class CondimentDecorator implements Beverage {
    protected Beverage beverage;
    
    public CondimentDecorator(Beverage beverage) {
        this.beverage = beverage;
    }
}

// 具体装饰器：牛奶
public class Milk extends CondimentDecorator {
    public Milk(Beverage beverage) {
        super(beverage);
    }
    
    @Override
    public String getDescription() {
        return beverage.getDescription() + ", 牛奶";
    }
    
    @Override
    public double cost() {
        return beverage.cost() + 5.0;
    }
}

// 具体装饰器：摩卡
public class Mocha extends CondimentDecorator {
    public Mocha(Beverage beverage) {
        super(beverage);
    }
    
    @Override
    public String getDescription() {
        return beverage.getDescription() + ", 摩卡";
    }
    
    @Override
    public double cost() {
        return beverage.cost() + 8.0;
    }
}

// 使用示例
public class CoffeeShop {
    public static void main(String[] args) {
        // 基础饮料
        Beverage espresso = new Espresso();
        System.out.println(espresso.getDescription() + " ￥" + espresso.cost());
        
        // 加牛奶
        Beverage espressoWithMilk = new Milk(espresso);
        System.out.println(espressoWithMilk.getDescription() + " ￥" + espressoWithMilk.cost());
        
        // 加牛奶和摩卡
        Beverage fancyCoffee = new Mocha(new Milk(new Espresso()));
        System.out.println(fancyCoffee.getDescription() + " ￥" + fancyCoffee.cost());
    }
}
```

## super(beverage)

### 1. 基本含义
```java
public CondimentDecorator(Beverage beverage) {
this.beverage = beverage;
}

public class Milk extends CondimentDecorator {
public Milk(Beverage beverage) {
super(beverage);// 调用父类构造器
}
}
```

**`super(beverage)`的作用是：**
- 调用父类`CondimentDecorator`的构造器
- 将传入的`beverage`参数传递给父类构造器
- 父类构造器将`beverage`保存到`this.beverage`字段中

### 2. 为什么需要这个？
```java
// 如果没有super调用，代码会变成这样：
public class Milk extends CondimentDecorator {
public Milk(Beverage beverage) {
// 自动调用父类无参构造器 super();
// 但父类没有无参构造器，会编译错误！
// 必须显式调用有参构造器
}
}
```

### 3. 完整的调用链示例
```
// 创建过程：
Beverage coffee = new Milk(new Espresso());

// 实际执行顺序：
1. new Espresso() → 创建Espresso实例
2. new Milk(espresso) →
a. 进入Milk构造器，调用super(beverage)
b. 进入CondimentDecorator构造器，设置this.beverage = espresso
c. 返回Milk实例
3. 最终Milk实例内部持有了Espresso的引用
```

## 二、new Mocha(new Milk(new Espresso()))

### 1. 逐层分解
```
// 从内到外创建：
1. new Espresso()// 最内层：基础咖啡
↓
2. new Milk(espresso)// 中层：加牛奶
↓
3. new Mocha(milkWithEspresso)// 最外层：加摩卡

// 相当于：
Espresso base = new Espresso();// 基础咖啡 ￥25
Beverage withMilk = new Milk(base);// 加牛奶￥25+5=30
Beverage withMocha = new Mocha(withMilk); // 再加摩卡 ￥30+8=38
```

```

fancyCoffee.cost()
↓
Mocha.cost()
↓ 调用 beverage.cost()// beverage是Milk实例
↓
Milk.cost()
↓ 调用 beverage.cost()// beverage是Espresso实例
↓
Espresso.cost() 返回 25
↓
Milk.cost() 返回 25 + 5 = 30
↓
Mocha.cost() 返回 30 + 8 = 38

// 实际的计算顺序：25 → 30 → 38（从内到外累加）
```

### 4. 装饰器的"洋葱"模型
```
// 可以想象成一层层包裹：
洋葱最中心：Espresso（核心功能）
第一层皮  ：Milk（添加牛奶功能）
第二层皮  ：Mocha（添加摩卡功能）

// 调用时从外向内穿透：
Mocha → Milk → Espresso
```

### 7. 关键理解点

1. **组合关系**：每个装饰器都持有一个`Beverage`引用
2. **链式调用**：方法调用沿着装饰链传递
3. **透明性**：装饰后的对象可以当作普通`Beverage`使用
4. **灵活性**：可以随意组合装饰顺序

这种设计模式的美妙之处在于：**可以无限嵌套装饰器**，每个装饰器只关心自己的功能增强，不需要知道其他装饰器的存在。
