1. 项目定位与背景

3. Maven 多模块结构详解

debugservice/                        ← 聚合根（pom）
├── debugservice-api/                ← 公共接口定义层（v1.0.19，独立发布）
│   └── 枚举、常量、序列化工具、ES 数据模型、DTO
├── debugservice-dao/                ← 数据访问层
│   └── MyBatis Plus + Zebra 连接池，2张核心表
├── debugservice-sdk/                ← Thrift SDK（v1.0.22，独立发布）
│   └── Thrift 接口定义、图策略客户端
├── debugservice-service/            ← 核心业务逻辑层
│   └── 请求处理、结果渲染、第三方集成、JTest

▎ 设计特点：api 和 sdk 单独定版发布，供其他服务依赖；service 和 dao 不单独发版，打包进 server。

4. 核心功能模块

4.1 图策略（DAG）调试 — 最核心功能

- 接收图策略执行请求，记录完整的请求/响应到 tbl_graph_debug_request
- 支持算子级别追踪，存入 tbl_graph_debug_operator
- 支持请求体 BLOB + Gzip 双路压缩存储（大请求场景）
- GraphTypeEnum、DebugLevelEnum 控制调试粒度

4.2 Octo RPC 调试

- OctoControllerV2 提供对任意 Thrift 服务的调用模拟
- 支持 v1/v2 两套接口，兼容历史版本

4.3 推荐链路回放（thirdparty 层）

集成了完整的美团外卖推荐链路：
召回 → RecHub → River → 排序 → Libra/商购排序 → 过滤 → CBase/MIG

4.4 JTest 单元测试辅助

- JtestController + JTestManageController
- JavaParser 解析源码，自动生成测试用例
- 支持测试管理和批量执行

4.5 结果可视化渲染

- ResultDisplayController + render/ 服务
- 将算法调试结果以可读格式展示

4.6 消息通知

- MessageController + message/ 服务
- 结合 Mafka 消息队列，支持异步通知

4.7 工具集（ToolsController）

- LionConfig 配置读取（LionController）
- POI/SPU 数据查询（PoiSpuController）
- Excel 导出（Apache POI）

6. 数据模型设计

tbl_graph_debug_request（调试请求主表）

核心字段分组：
├── 定位字段：traceId, appKey, scene, graphKey, version
├── 服务信息：hostName, hostIp, servicePort, serviceName, methodName
├── 请求存储：requestString / requestBinary(BLOB) / requestStringCompressed
├── 响应存储：responseString / responseStringCompressed
├── 业务调试：requestBusiDebug / requestBusiDebugCompressed
├── 用户身份：userId, mis, uuid
├── 状态跟踪：status, exceptionInfo, memo
└── 时间/链路：createTime, span(mtrace spanId)

▎ 设计亮点：同时支持文本、二进制、Gzip 三种存储形式，适配不同大小的请求体。

7. 关键设计特点

7.1 双版本 API（v1 / v2）

- controller/ vs v2/controller/，兼容历史调用方
- v2 整体更规范，功能更完整

7.2 请求路由（RouteService）

- 支持将调试请求路由到指定机器/环境
- 是"回放"能力的核心基础设施

7.3 Bean 定义覆盖

spring.main.allow-bean-definition-overriding=true
主动允许 Bean 覆盖，表明存在多环境配置差异化需求。

7.4 异步能力

@EnableAsync   // 支持 @Async 方法
配合 XFrame ThreadPool 线程池，用于非阻塞调试记录写入。

7.5 groupId 不一致（潜在问题）

- 根项目：com.sankuai.worldtree
- debugservice-service：com.markus（历史遗留，建议统一）

10. 一句话总结

wtdebugservice 是美团外卖推荐系统的核心调试平台，采用标准的 Spring Boot 多模块架构，通过 Thrift RPC 深度集成 RecHub / Libra / River 等推荐引擎，为算法工程师提供图策略、算子、Octo RPC 的全链路调试能力，是外卖推荐研发效率的关键基础设施。
